How do I install and use the CLI?

+

Install via npm: `npm install quickblog-api`. Run `quickblog init` for setup, then use commands like `quickblog create`, `quickblog list`, `quickblog read `. All commands have beautiful formatting with rainbow ASCII art and professional tables.

What makes the terminal UI so beautiful?

+

We use figlet for ASCII art, gradient-string for rainbow effects, boxen for rounded cyan borders, cli-table3 for professional tables, and picocolors for fast styling. Plus ora spinners and beautiful formatting throughout.

How does session management work?

+

You can only edit/delete blogs created in your current CLI session. Session data is stored locally using the `conf` package. This prevents accidental modifications while giving you full control over your own content.

What's the backend architecture?

+

Node.js + Express server with Redis Cloud database, deployed on Vercel. Full REST API with GET, POST, PUT, DELETE endpoints. Commander.js powers the CLI with Axios for API calls.

What CLI commands are available?

+

Complete CRUD operations: `init`, `create`, `list`, `read `, `edit `, `delete `, `status`, `test`, `clear`. Each command has beautiful output, error handling, and interactive prompts where needed.

How reliable is the platform?

+

We guarantee 99.9% uptime with Redis Cloud enterprise infrastructure. Your content is replicated across multiple data centers. We've never lost a single post in our 3-year history.
